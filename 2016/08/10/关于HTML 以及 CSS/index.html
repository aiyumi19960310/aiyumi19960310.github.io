自己看了很多有关于面试方便的资料，也总结了一些面试的题目，现在将一些比较常见的，也是每一个程序猿必备的只是做一个简单的分享。

一. 关于HTML 以及 CSS

1、作为一个前端程序猿(毁天灭地，orz)，那么我们一定对于各种浏览器特别熟悉，这个时候在面试的时候就会出现下面这种问题：

  • 请说出开发中常用的浏览器，以及他的内核。
    ps (作为一个前端程序猿) 小意思

    1. 微软公司 使用的 IE 浏览器，内核为 trident，现在 IE 已经换成了 EG
    2. 由 mozilla 开发的 FireFox ， 内核使用的是 gecko
    3. 苹果开发的 Safari ， 使用的是 webkit
    4. 最后是 Chrome 开发的 基于 webkit 的 Blink

 

2、对于 IE 浏览器，我们知道有 怪异模式(Quirks) 以及 标准模式(Standards),那么他们两者之间有什么区别？

  • 首先我们要知道，为什么会产生这种怪异模式，那么在 IE6 之前的时候， css还没有完全成熟，因此 IE5 之前对 css 的支持特别的差，而到了 IE6 对 CSS 的支持更好了，但是很多网页使用的是之前的布局，如果使用 IE6 的 CSS 那么页面的布局就会产生新的变化，即页面显示不正常。所以在网页的头部区域增加了一个 DTD 的声明 表示。如果使用的 DTD 声明 表示使用 新的 CSS 来渲染页面， 如果没有使用那么就是用之前的布局模式。这种布局被称为怪异模式。

  • 区别：

    总体上会有布局、样式解析、脚本执行上的三个区别

    1. 盒模型:在W3C的标准中，如果设置了一个元素的宽高，指的是盒子内容的宽高，而在怪异模式下，设置的宽高包括了padding、border。
    2. 设置行内元素高度: 在标准模式中，给span等行内元素设置 width和 height都不会生效，而在怪异模式中会起效
    3. 设置百分比的高度: 在标准模式下，一个元素的高度是由其包含的内容决定的，如果父元素没有设置百分比高度，子元素设置一个百分比高度是无效的。
    4. 使用使用margin:o auto可以使盒子水平居中，但是在怪异模式下无效
    5. @improtant在IE7以上及其他浏览器识别，但是IE6以下都无效
    6. …….

当然了还有很多其他的区别，这些就需要自己去了解了，打出什么不重要，重要的是，不要虚！不要虚！不要虚！

 

3、在网页开发中，我们经常会碰见不同的浏览器对于新的语法的兼容性较差的问题，那么我们在构建项目的时候，是先将页面设计成最完整的功能，还是先实现基本需求呢？这就涉及到了渐进增强，以及优雅降级。

  • 渐进增强

    渐进增强就是，在开发过程中，针对低版本的语法兼容差的浏览器进行基础开发，实现基本功能，在基本功能搞定之后，针对高版本的浏览器进行效果、交互、等改进和追加的功能以达到更好的用户体验

    • 渐进增强认为应该关注的是内容的本身

  • 优雅降级

    优雅降级则是与之相反的，在最开始开发的时候，就做最完整的功能，然后再后期开发中，逐步兼容低版本浏览，适当的降低用户体验，但是最后也是要保留基本功能在低版本浏览器上。一般来说，降级是针对主流浏览器的低版本进行兼容的。

    • 优雅降级则认为针对高级的浏览器设计，后期兼容低版本主流浏览器

 

4、对于 HTML 中的标签语义化，有什么样的理解。

  • 首先HTML本身是没有表现的，也就是说我们平时看到的 h 标签加粗，字体增大等都不是其表现，这些都是 CSS 的样式修饰的结果，所以当样式表丢失或者去掉样式的时候，能让页面呈现清晰的结构分层。但是，浏览器都有默认样式，默认样式只是为了更好的表达标签的语义化。可以说默认样式和标签的语义化是不可分割的。
  • HTML语义化的优势在于，搜索引擎的爬虫依赖标记来确定上下文和关键字的权重。搜索引擎可能会忽略掉你的CSS表现形式，而不会忽略你的HTML语义标签。另外标签语义化也利于CSS以及JavaScript进行编写。

 

5、上面说到了HTML标签的语义化，那么与之相应的还有网页的 SEO 搜索优化，这是一个前端工作者必须要掌握的知识。

  • 简化代码结构，利于搜索引擎抓取:

    • 页面尽量采用DIV+CSS编写，table代码结构臃肿，而div和CSS的结构更加的简单清晰
    • 搜索的js和css采用外链方式，使用table可能会产生一些css样式上的问题。
    • 图片采用精灵图的方式，尽量减少请求次数
      注意:使用表格table还是比div方便,但是随着而来的会有各种问题
    • 下面两段代码就可以看出 简单的HTML结构比table更加适合页面的编写

  • 重要内容优先加载（第一个链接最好是网站主关键词，不刻意要求），可以用css来处理，索引一篇文章的长度也是有限制的，一定要把最重要的内容，优先展现给蜘蛛，这方面你可以通过查看一些比较大的网页快照来求证。

  • 每个页面只能出现一次H1标签，H2标签可以多次：H1权重很高，普遍认为仅次于title，一般资讯详情页的标题、商品详情页的标题，都放在H1里。

  • 图片一定要添加alt属性，title属性可选：蜘蛛不认识图片上的内容，只能通过alt属性来判断，如果是商品列表页，所有商品都加了alt和title的话，容易造成堆砌关键词，所以我一般是只加alt属性。

  • 图片大小声明：如果图片大小不做定义的话，页面需要重新渲染，就会影响到速度。

  • 链接可根据实际需求添加title属性以及nofllow值，非特殊性链接，链接地址一定要写入herf属性，有些前端开发人员为了省事，直接用div加个click事件当链接，在视觉上和使用上确实是实现了链接效果，但是做过SEO优化的人员都知道，蜘蛛目前对于js的支持很差，基本无法读取里面的链接地址。所以说用click事件是绝对不允许的，特别是一些重要的导航链接。

  • 页面内容尽量不要做成flash、图片、视频:这些东西蜘蛛是抓不到的，就算是必须的，也要生成相应的静态页面。有很多企业站看着很炫，全站flash，老板看着是爽了，做SEO优化的人员就要抓狂了，全站没一个链接。

  • 除首页外别的页面最好要加上面包屑型导航，导航结构一定要清晰。

  • 做好404页面，一般会加首页链接及错误提示，并测试其返回状态码为404：

    • 1、用户体验友好，可以留住用户，不至于直接关闭页面；
    • 2、蜘蛛友好，可以返回抓取其他页面。

  • 网站结构呈扁平状树型，目录结构不宜过深，每个页面离首页最多点击不超过3次，过深不利于搜索引擎的抓取。
