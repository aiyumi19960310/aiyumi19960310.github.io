关于兼容性问题

  • JavaScript html css
兼容
  view

在工作当中，经常云间许多兼容性的问题，不管是PC端还是web移动端，由于各种浏览器的厂商实现各种规范的方式不同，以及对ES5、ES6支持等问题，导致了各种的样式失效，html结构不同，js的代码不同等等都是兼容的问题。这时候就有 CSS HACK 的说法，指的是，对于不同的浏览器写出不同的 CSS兼容样式表

对于面试中的提问

  • 在面试当中，经常和会被提问到:请说说你对兼容性的看法，或者是请谈谈你曾经遇到的兼容的问题。但是在实际上，没有多少人会去死记硬背有多少个兼容性的问题，只有在写到这个地方的时候，才会反映过来，这个地方有坑，需要跳过。或者说，如果忘记了，打开搜索就有了。

  • 大部分时候我们需要应答的并不是兼容性的问题，而是考察是否对于这个兼容性的问题是否有认知，这种兼容性的问题是怎么出现的。

    • 1、兼容的根本就是因为不同的浏览器对于语法不同的支持产生的，其中最严重的就是IE浏览器

    • 还有记住一点:IE浏览器中的各种兼容性问题，根本原因就需要延伸到javascript编写出来的时候发生的事情了，但是ie的兼容性问题基本上能用这几个方法解决display:inline、font-size:0、float等方法解决。剩余的其他问题就在于IE浏览器对于不同API的不同的命名。最简单的在js中，其他的浏览器的写法是.click等事件，但是在IE中前面加了个on。这些的问题就在javascript在编写出来的时候问了抢占市场产生的问题了。

 

普遍存在的兼容性问题

  • 针对ie存在的普遍兼容问题
    • 1、双边距BUG float 引起的 解决方法:使用display:none
    • 2、3像素的问题:ie中float之后文本之间存在3像素的差距， float 引起 解决方法 display:none
    • 3、超链接 hover 点击后失效 牢记 love hate 原则 即 link、visited、hover、active
    • 4、ie z-index 问题给父元素添加 position:relative
    • 5、png24 在ie上出现背景，改用 png8 如果实在不行就该用js更改
    • 6、js中 事件处理对象 在 ie中只能使用 event 在firefox可以使用e，问了兼容进行判断var e = e ? e : event
    • 7、ie中可以直接.自定义属性名进行获取，在Firefox中需要使用getattribute进行获取 但是ie也支持这个写法 因此同意使用该方法即可
    • 等等….还存在其他一些ie的兼容性问题，在这不一一赘述，如果想要了解那么 43种兼容问题

  • 关于一些通用的兼容问题

    • 1、在多个浏览器中存在初始样式，如padding、margin等，解决办法是在写css样式表的时候加上 * {padding:0;margin:0}但是不推荐这么写，因为会影响到性能，存在很多用不到的标签样式也会被初始化，推荐需要哪个标签就写上哪个标签就行了
    • 2、在chrome浏览器中如果字体样式小于12px那么会自动提升到12px，这个时候需要加一小段代码-webkit-text-size-adjust: none;，这样就能解决这个问题。

    • 3、兼容不同的浏览器还有一种就是，同一个效果不同浏览器的api不同这个时候就有了浏览器的私有前缀 火狐Firefox: -moz-; 苹果Safari: -webkit-; 欧朋Opera: -o-; IE浏览器IE: -ms-，对于IE浏览器还有专用的写法

 

最后的思考

  • 在这个网络发展的大时代里，不要一味的追求兼容问题，毕竟代码需要进步，那么就需要甩掉沉重的历史冗余代码。要学会取舍，当然，针对于不同的客户群体有不同的兼容代码。牢记一点，应变，毕竟代码有不同的写法。最后，记住兼容查询的网站。例如:can i use。或者在编辑器中下载一些插件。
