javascript设计模式

  • JavaScript
模式
  view

在其他语言中存在这设计模式，慢慢的javascript的需求提高之后，也有了很多的设计模式，我在这里了解了几种设计模式。

单例模式（常见、常用）

  • 在我的理解当中，单例模式即是如果我有一个需求，那么将这个需求实现并且使之能够被复用(这里的复用指的是同样的情况下使用该代码)，即只能满足一种需求的函数。

简单工厂模式（常见、常用）

  • 首先，我们明白一件事情，工厂模式是什么，即一个通过创建一个函数，将这个函数进行实例化，对传入不同的参数产生不同的实例化对象。

观察者模式（重要）

  • 观察者模式是日常开发中最常用的，我对他的理解即是，有一件事情，这个事情由别人是监控，这就是观察者模式。在这个模式中存在”发布者”和”订阅者”，被监控的对象即是”订阅者”，这个监控人就是”发布者”。
  • 在实际开发中，观察者模式可以很好的对耦合的代码进行解耦。

适配器模式

  • 适配器模式实际上，就是将一个不符合我的格式的函数，将其转换成为符合我的需求格式的函数。即jquery的库，如果说你并不想学习这个库，但是你想用简洁的语法，那么你可以使用一个简单的小封装$id = function( id ){return jQuery( '#' + id )[0];}，将开始的id转换成jquery的$id以满足需求

代理模式(委托)

  • 代理模式是最容易被理解的一种模式，将自己需要办的事情，让别人去完成就是代理。

桥接模式

  • 个人的理解就像mvc思想一样，将表现层和数据层进行抽离，但是又能将之结合，每次只需要修改或重写一边就能实现功能了。

外观模式（门面模式）

  • 外观模式就是将几个需要使用的函数进行包装调用，直接返回函数执行的结果。

  • 最简单的情况就是经常使用的冒泡和阻止默认事件
